Architecture Overview

This document provides a high-level overview of the project's structure and responsibilities.

Repository layout:
- `docs/` — documentation files
- `scripts/` — helper scripts for setup, build, run, and test
- Root files: `README.md`, `LICENSE`

Design goals:
- Keep scripts minimal and reproducible.
- Documentation should be the source of truth for workflows.

If you add new modules or services, extend this document with diagrams or component descriptions.
