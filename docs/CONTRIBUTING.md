Contributing

Thanks for wanting to contribute!

Quick workflow:
1. Fork the repository and create a branch named `feature/your-change`.
2. Make small commits and include tests where appropriate.
3. Update or add documentation in the `docs/` folder.
4. Open a pull request and describe the change and rationale.

Style and tests:
- Keep changes focused and documented.
- Add tests when changing behavior; document manual verification steps if tests are not possible.

Communication:
- Link related issues in your PR. Use descriptive titles and clear summaries.
